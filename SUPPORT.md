# LayerDraw Support

LayerDraw is pre-release and does not yet provide a supported installation or production service-level agreement. Use the channel that matches the request so maintainers can handle it with the correct visibility and security.

## Questions and Ideas

Use [GitHub Discussions](https://github.com/dencyuinc/layerdraw/discussions) for usage questions, design questions, early ideas, and community conversation. Search existing discussions and documentation before starting a new topic.

## Bugs and Proposals

Use the repository's structured [Issue Forms](https://github.com/dencyuinc/layerdraw/issues/new/choose):

- Bug Report for reproducible defects;
- Feature Request for bounded product behavior;
- Design Issue or RFC for contract and architecture changes;
- Storage Provider for storage adapter work;
- Registry Content for packs and templates; and
- Security Guidance for public defense-in-depth documentation that does not disclose a vulnerability.

Do not use an issue as a general support ticket or publish customer data, credentials, private deployment configuration, or exploit details.

## Security Vulnerabilities

Follow [SECURITY.md](SECURITY.md). Never report a vulnerability with sensitive details through a public issue, Discussion, or Code of Conduct address.

## Conduct Reports

Report Code of Conduct violations privately as described in [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md). The conduct address is not a product support or security channel.

## Licensing and Commercial Requests

Review the [LayerDraw License Design](docs/legal/README.md) and [license use cases](docs/legal/use-cases.md) first. Questions about Commercial / OEM licensing, trademarks, or a proposed hosted offering should use the [株式会社DENCYU contact form](https://dencyu.co.jp/contact/), not a public issue.

## Response Expectations

Community channels have no guaranteed response or resolution time. Security reports and separately contracted commercial support follow their applicable policies or agreements.
